## Key Insights & Lessons Learned

### Technical Insights

1. **LIKE reversal is a platform pattern.** `? LIKE pattern_column` (where the pattern is stored in the DB column and the value is the query parameter) enables wildcard dataset matching from config tables. Established in CheckFactory (story 2.1), reused in EnrichmentResolver (story 2.4). Future stories querying pattern tables must use this reversal.

2. **Baseline providers need centralization.** Stories 2.5–2.8 each independently implemented `BaselineProvider`, `NoOpBaselineProvider`, and `JdbcBaselineProvider` with the same contract. This works but creates drift risk. Epic 3+ should consolidate to a shared baseline query utility.

3. **JDBC safety checklist for Java JDBC code.** Three JDBC safety patterns were missed and caught in review for story 2.10:
   - Save and restore `autoCommit` state in finally block
   - Add rollback exceptions as suppressed to the primary exception (not swallow them)
   - Use `TypeReference<Map<String,Object>>` (not raw `Map.class`) for JSON deserialization

4. **H2 DECIMAL precision gotcha.** `DECIMAL` without explicit precision/scale in H2 2.2.224 defaults to scale=0. Use `DECIMAL(20,5)` (or appropriate precision) in H2 DDL for any numeric column that stores fractional values. This does not affect Postgres.

5. **spark-avro is provided in Spark 3.5 but requires test-scope dependency for local tests.** `spark-sql` (provided scope) does not transitively include `spark-avro`. For local Avro test fixture support, add `spark-avro_2.13:3.5.0` with `test` scope. The fat JAR is unaffected — cluster has it bundled.

### Process Insights

6. **A commitment without an artifact is just a wish.** The Epic 1 pre-review checklist committed to as a "before Epic 2 Story 1" action item was never created as an actual file. Stale TDD comments recurred as a direct result. Action items must produce concrete artifacts, not just intentions.

7. **Constructor validation is load-bearing.** Multiple stories had null/blank guard gaps surfaced in review (2.1: MetricNumeric, MetricDetail; 2.9: constructor null-guards). The rule is in project-context.md. It needs to be on a checklist that developers actually run before submitting for review.

8. **ATDD red-phase is valuable.** Stories 2.5–2.9 used the ATDD workflow (tests written before implementation). These stories had fewer implementation surprises and cleaner first-pass implementations than the infrastructure stories.

9. **DqsJob config loading must be explicit.** Story 2.4 review found DqsJob wasn't loading `application.properties` from classpath. DB config must be explicitly loaded — never assumed from environment. Add this to the Java component integration checklist.

### Team Insights

10. **Zero-regression discipline scaled to 10 stories.** The full regression suite ran after every story. With 167 tests spanning scanner, reader, enrichment, 5 checks, and BatchWriter — no story broke any previous story's tests. Cumulative build with full regression works at scale.

11. **Pattern propagation works without coordination.** BaselineProvider (2.5), LIKE reversal (2.1→2.4), per-dataset isolation (2.3→all) — patterns coined in one story propagated to subsequent stories without explicit mandates. Good patterns spread when the first implementation is clean.
