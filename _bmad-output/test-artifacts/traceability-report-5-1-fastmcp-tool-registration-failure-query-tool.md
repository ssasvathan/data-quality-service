## Story Summary

LLM consumers need a way to ask "What failed last night?" and get a structured failure summary without opening the dashboard. Story 5.1 creates the MCP layer on top of the existing `dqs-serve` FastAPI app using FastMCP 3.2.0. The `query_failures` tool queries `v_dq_run_active` and returns formatted text for LLM consumption.

**Acceptance Criteria:**
- **AC1:** FastMCP tools registered and discoverable by LLM clients when serve starts
- **AC2:** Failure query tool returns formatted text: count, dataset names, check types, LOB grouping
- **AC3:** No failures → "All datasets passed in the latest run ({timestamp}). {N} datasets processed."
