# Spark Job Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the DQS Spark application (Scala) that reads partitioned data, dynamically generates data quality checks from a JSON configuration, executes them, and logs the metadata back to the shared Postgres database.

**Architecture:** A standard `sbt`-based Scala project. It runs a single batch `SparkSession` that reads input metrics, evaluates threshold rules defined in a config object, and pushes JDBC dataframes into the `dataset`, `dq_run`, `dq_check_result`, and `dq_metric_snapshot` tables.

**Tech Stack:** Scala 2.13, Apache Spark 3.5.0, PostgreSQL JDBC Driver, uPickle (for JSON parsing), ScalaTest

```scala
package com.dqs.config

import upickle.default._

case class CheckConfig(checkType: String, min: Option[Int] = None)
object CheckConfig {
  implicit val rw: ReadWriter[CheckConfig] = macroRW
}

case class RuleConfig(dataset: String, checks: Seq[CheckConfig])
object RuleConfig {
  implicit val rw: ReadWriter[RuleConfig] = macroRW
  
  def fromJson(json: String): RuleConfig = read[RuleConfig](json)
}
```

```scala
// spark-job/src/main/scala/com/dqs/db/PostgresWriter.scala
package com.dqs.db

import java.util.Properties

class PostgresWriter(host: String, port: Int, db: String, user: String, pass: String) {
  val jdbcUrl: String = s"jdbc:postgresql://$host:$port/$db"
  
  def connectionProperties(): Properties = {
    val props = new Properties()
    props.setProperty("user", user)
    props.setProperty("password", pass)
    props.setProperty("driver", "org.postgresql.Driver")
    props
  }
}
```
